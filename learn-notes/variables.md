# Rust 变量绑定详解

> 对应项目：`projects/02-variables`
> 掌握目标：理解 Rust 中所有变量声明方式及其差异

---

## 一、核心哲学：不可变优先

Rust 变量与大多数语言最大的不同：**默认不可变**。

```rust
let x = 5;     // x 是不可变的
x = 6;         // ❌ 编译错误！
```

这不是bug，是**刻意设计**。为什么？

### 不可变的好处

| 好处 | 说明 |
|------|------|
| **并发安全** | 不可变数据天然线程安全，不需要锁 |
| **可推理** | 读代码时，`x` 的值永远不会"惊喜地"改变 |
| **编译器优化** | 不可变值可以被复制、内联、重排而不用担心副作用 |
| **减少 Bug** | 一个函数不会悄悄修改你传进去的值 |

## 二、`let` 语句：变量的诞生

```rust
let 变量名 [: 类型] = 初始值;
```

### 类型推断

```rust
let x = 42;         // i32（整数默认类型）
let y = 3.14;       // f64（浮点默认类型）
let z = true;       // bool
let c = 'A';        // char
let s = "hello";    // &str（字符串切片，存在静态区）
```

Rust 的类型推断非常强大，绝大多数情况下不需要手写类型。但有几种情况**必须标注**：

```rust
// 1. 无法推断目标类型时
let num: i32 = "42".parse().unwrap();   // parse 不知道你要什么类型

// 2. 涡轮鱼语法（不标注也能用）
let num = "42".parse::<i32>().unwrap(); // ::<> 就是"涡轮鱼"

// 3. const / static 永远必须标注
const MAX: u32 = 100;
```

### 未初始化？不行！

```rust
let x;          // ❌ 编译器要求初始化
// Rust 不允许"先声明后赋值"，必须声明时给值
let x = 5;      // ✅
```

这一点和 C/C++ 完全不同。在 Rust 中**不可能**有未初始化的变量。

### 延迟初始化只有一种方式

```rust
let x;              // 先声明（类型需要可推断）
if condition {
    x = 1;
} else {
    x = 2;
}
// x 在这里可以安全使用
println!("{}", x);
```

编译器会**静态分析**所有分支，确保 `x` 在所有路径上都被初始化了且只被初始化一次。如果某条路径漏了，编译不过。

## 三、`mut`：请求可变性

```rust
let mut counter = 0;     // 显式声明"这个变量可以变"
counter += 1;            // ✅
counter = 10;            // ✅
// counter = "hello";    // ❌ 即使 mut，类型也不能变！
```

### mut 的限制

Rust 的 `mut` 是**严格类型的可变**：

```rust
let mut x = 5;
x = 10;        // ✅ 值变了
// x = "hi";   // ❌ 类型没变，不能改类型
```

| 可以改 | 不可以改 |
|--------|----------|
| 变量的**值** | 变量的**类型** |

`mut` 还标记了**所有权借用**的能力——只有 `mut` 的变量才能被 `&mut` 借用（后面所有权章节会讲）。

## 四、Shadowing（变量遮蔽）

Rust 允许用 `let` 重复声明同名变量，**新变量会"遮蔽"旧变量**：

```rust
let x = 5;          // x = 5 (i32)
let x = x + 1;      // x = 6 (i32) — 新 x 遮蔽旧 x
let x = "hello";    // x = "hello" (&str) — 类型都变了！
```

### Shadowing vs mut

| | `let mut` + 赋值 | `let` 遮蔽 |
|---|---|---|
| 改变值 | ✅ | ✅（新绑定） |
| 改变类型 | ❌ | ✅ |
| 需要 `mut` | ✅ | ❌ |
| 本质 | 修改内存中的同一个位置 | 创建一个新变量，旧变量被隐藏 |
| 旧值 | 被覆盖 | 仍然存在（被隐藏但没被销毁*） |

*如果旧变量被引用持有，它不会被销毁，直到引用消失。

### 作用域中的 Shadowing

```rust
let x = 1;
{
    let x = 100;     // 只在内层作用域遮蔽
    println!("{}", x); // 100
}
println!("{}", x);     // 1 — 外层不受影响
```

## 五、`const`：编译时常量

```rust
const MAX_POINTS: u32 = 100_000;
const PI: f64 = 3.141592653589793;
const APP_NAME: &str = "MyApp";
```

### const 的特点

| 特性 | 说明 |
|------|------|
| **必须标注类型** | 编译器不推断 const 的类型 |
| **编译时求值** | 值必须是编译时就能算出来的表达式 |
| **无固定地址** | const 在使用处**内联展开**，就像 `#define` 宏 |
| **任何作用域** | 可以放在全局、模块、函数、甚至 `impl` 块 |
| **永远不可变** | mut const 不存在 |

```rust
// 合法的 const 表达式：
const SECONDS: u32 = 60 * 60 * 24;    // 编译时运算
const HALF: f64 = PI / 2.0;           // 引用其他 const
const ARR: [i32; 3] = [1, 2, 3];     // 数组

// 非法的：
// const NOW: u64 = get_timestamp();  // ❌ 函数调用不是编译时
// const VEC: Vec<i32> = vec![1,2,3]; // ❌ Vec 分配堆内存，编译时不行
```

## 六、`static`：全局静态变量

```rust
static LANGUAGE: &str = "Rust";        // 不可变 static
static mut COUNTER: u32 = 0;           // 可变 static（需要 unsafe）
```

### const vs static

| | const | static |
|---|---|---|
| 内存地址 | 没有（内联展开） | 有固定地址 |
| 生命周期 | 使用时求值 | `'static`（程序全程存在） |
| 修改 | 不可能 | unsafe 下可以 |
| 使用场景 | 魔法数字、配置常量 | 全局状态、FFI 导出 |

```rust
// static 有固定地址——可以做原始指针
let addr: *const u32 = &LANGUAGE as *const &str as *const u32;
// const 做不到——它没有地址
```

**实践中**: 99% 的情况用 `const`。只有做 FFI、嵌入式开发或真需要固定内存地址时才用 `static`。

## 七、解构绑定

`let` 不仅能绑定一个名字，还能**一次性拆开**复合类型：

```rust
// 元组解构
let pair = (10, 20);
let (a, b) = pair;           // a = 10, b = 20

// 嵌套解构
let nested = ((1, 2), 3);
let ((x, y), z) = nested;    // x = 1, y = 2, z = 3

// 忽略部分值
let (first, _, third) = (1, 2, 3);   // 只取 1 和 3

// .. 忽略剩余
let (head, .., tail) = (1, 2, 3, 4, 5);  // head = 1, tail = 5
```

后续学到枚举、结构体时，解构能力会更丰富。

## 八、作用域规则

```rust
{                           // ← 作用域开始
    let x = 5;              // x 从这一行开始有效
    // x 可用
}                           // ← x 被 drop，不再存在
```

### Drop 顺序

变量按**声明的相反顺序**被 drop：

```rust
let a = String::from("first");
let b = String::from("second");
let c = String::from("third");
// drop 顺序: c → b → a（后进先出，栈式）
```

## 九、一张表总结

```
┌──────────┬──────────┬──────────┬──────────┬────────────┐
│  机制     │ 可以改值  │ 可以改类型 │ 需要类型标注│ 求值时机   │
├──────────┼──────────┼──────────┼──────────┼────────────┤
│ let      │ ❌       │ ❌       │ 可选      │ 运行时     │
│ let mut  │ ✅       │ ❌       │ 可选      │ 运行时     │
│ 遮蔽 let  │ ✅(新绑定)│ ✅       │ 可选      │ 运行时     │
│ const    │ ❌       │ ❌       │ 必须      │ 编译时     │
│ static   │ ❌(*)    │ ❌       │ 必须      │ 编译时     │
└──────────┴──────────┴──────────┴──────────┴────────────┘

(*) static mut 存在但需要 unsafe
```

## 十、新手常见误区

**误区 1**：`let mut` 和 `let` 遮蔽是一回事
→ 不是。`mut` 原地修改，遮蔽创建新绑定，可以改类型。

**误区 2**：Rust 的不可变变量和 C 的 const 一样
→ 不同。Rust 的 `let` 不可变是**默认的、深度的**。C 的 const 只是"不能通过这个名字改"，其他人可以通过指针改。

**误区 3**：变量必须先声明类型
→ 不需要。Rust 的类型推断很强大，大部分时候省略即可。

**误区 4**：`const` 和 `let` 只是"能不能改"的区别
→ 远不止。const 是编译时内联展开的无地址值，let 是运行时分配的有地址绑定。二者的行为模型完全不同。
